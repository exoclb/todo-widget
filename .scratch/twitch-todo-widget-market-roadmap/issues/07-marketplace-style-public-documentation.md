## What to build

Create marketplace-style public documentation that helps streamers understand, evaluate, install, and configure the widget. The documentation should present the widget as a polished StreamElements product with screenshots or demo media, clear feature status, setup instructions, and example configurations.

## Acceptance criteria

- [ ] README shows the widget value proposition, supported features, and known limitations clearly.
- [ ] README includes screenshots or demo media that reflect the implemented visual presets and layout modes.
- [ ] README provides concise StreamElements installation steps for the four widget tabs and field configuration.
- [ ] README includes example configurations for at least two common stream styles.
- [ ] README documents preview usage, smoke test usage, and when to enable debug mode.
- [ ] Public documentation distinguishes implemented features from planned future features.
- [ ] Visual assets or screenshots are reviewed by a human before the issue is considered done.
